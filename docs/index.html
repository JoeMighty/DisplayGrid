Features

Everything you need. Nothing you don't.

Built for operators who need reliability, not complexity. Every feature earns its place.

Multi-Screen Zones

Organise displays into named zones. Manage dozens of screens from a single dashboard without losing track.

Live WebSocket Updates

Push playlist changes instantly. No polling, no page reloads — screens update the moment you save.

Offline Resilience

IndexedDB + Cache API keeps screens playing through network outages. Content survives a router reboot.

Kiosk Lock Screen

PIN-protected exit overlay. No cursor, no browser chrome, no way out without the code. Built for public spaces.

Role-Based Access

Super Admin, Admin, Operator, Viewer. Fine-grained control over who can edit, upload, and configure.

Emergency Override

Push urgent content to every screen instantly. One click replaces all active playlists site-wide.

Quick Start

Up and running in three steps.

A single install script handles dependencies, generates your secret key, and runs the database migration.

01

Clone & Install

git clone https://github.com/JoeMighty/DisplayGrid.git
cd DisplayGrid
bash scripts/install.sh

The install script checks for Node 20+, installs pnpm if needed, generates a secret key, and runs migrations.

02

Configure your environment

# Edit .env — set NEXTAUTH_URL to your LAN IP or local domain
# Never use localhost — use your machine's LAN IP or mDNS hostname

NEXTAUTH_URL=http://displaygrid.local:3000
NEXT_PUBLIC_WS_URL=ws://displaygrid.local:3001
# NEXTAUTH_SECRET is auto-generated by install.sh

Use your machine's hostname or LAN IP. Display clients connect over the network, not localhost.

03

Start the server & complete setup

pnpm dev

# Open http://displaygrid.local:3000 in your browser
# You'll be guided through a first-run setup wizard
# Create your admin account, configure your first screen
Requires Node.js 20+ and pnpm. Runs on Linux, macOS, and Windows. For production deployments, see the systemd service guide on GitHub.
Hardware

Runs on what you already have.

DisplayGrid has modest requirements. The server is just a Node process. The display client is just a browser tab.

Server

Dashboard · API · WebSocket · Database

MIN Raspberry Pi 4 (2GB RAM) — any always-on machine works. Linux, macOS, or Windows.
REC Raspberry Pi 5 (4GB+ RAM) with active cooling. Handles a large number of concurrent display clients comfortably.
NOTE The server doesn't need to be powerful — it just needs to be always-on and reachable from all display clients on the network.
:3000
Dashboard
:3001
WebSocket

Display Client

Any device that can run Chromium in kiosk mode

4K Raspberry Pi 5 — recommended for 4K content and smooth video playback.
1080p Raspberry Pi 4 handles 1080p content well, including images and web slides.
ALT Budget x86 mini PC, old laptop, Mac mini — any machine with Chromium works. No special GPU required for standard content.
